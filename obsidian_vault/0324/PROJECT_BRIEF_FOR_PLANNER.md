## 2. 已完成系統

- 8方向移動、走/跑狀態
- 近戰（劍）+ 法術（法杖）攻擊
- Dash 位移（無敵幀、Cancel攻擊）
- Hit Stop / 打擊感

### 敵人 AI ✅
| 敵人 | 類型 | Debug鍵 |
|-----|------|---------|
| Slime | 近戰追擊 | F6 |
| Archer | 遠程保持 | F7 |
| Boar | 突進爆發 | F8 |

- 武器：升級（0★→5★）、詞綴、符文槽
- 防具：頭盔/胸甲/護腿/靴子
- 符文石：12種（8普通+4核心），鑲嵌機制

### 存檔系統 ✅
- Save v6，含 checksum 驗證
- 完整狀態保存（位置/HP/裝備/背包/任務）

## 4. 需要企劃決定的內容

### A. 世界架構

**問題 1：遊戲世界如何組成？**

選項：
- [ ] **單一大地圖** — 無縫世界，無場景切換
- [ ] **Hub + 關卡制** — 中央安全區 + 傳送到各關卡
- [ ] **線性章節** — 關卡 1 → 關卡 2 → ...
- [ ] **開放區域** — 多個可互相連接的區域

**問題 2：目前需要幾個場景/區域？**

建議最小組合：
```
建議 A（最小可行）:
├── Town_Hub（安全區，有鐵匠）
├── Field_1（野外，敵人 Slime）
├── Field_2（野外，敵人 Archer/Boar）
└── Boss_Arena（Boss 戰）

建議 B（完整版）:
├── Town_Hub
├── Forest（森林，低等敵人）
├── Cave（洞穴，中等敵人）
├── Ruins（遺跡，高等敵人）
├── Boss_Dungeon（多房間地城）
└── Secret_Area（隱藏區域）
```

### B. 場景切換機制

**問題 3：如何切換場景？**

| 選項 | 說明 | 技術複雜度 |
|-----|------|----------|
| 走邊界切換 | 走到場景邊緣自動載入下一區 | 低 |
| 傳送門 | 與傳送點互動後載入 | 低 |
| 選單傳送 | 從地圖選單選擇目的地 | 中 |
| 無縫串流 | 動態載入/卸載區塊 | **高** |

### C. 存檔點設計

**問題 4：存檔機制？**

- [ ] **隨時存檔** — F5/F9 隨時可用（目前）
- [ ] **存檔點** — 必須找到營火/石碑才能存檔
- [ ] **自動存檔** — 進入新區域自動存檔

### D. Town Hub 設計

**問題 5：安全區需要哪些功能 NPC？**

目前已確定：
- ✅ 鐵匠（升級/符文/分解）

待決定：
- [ ] 商人（買賣物品）
- [ ] 任務發布板（接任務）
- [ ] 治療師（回復 HP）
- [ ] 倉庫管理員（額外儲存空間）
- [ ] 傳送師（快速移動）

### E. 關卡設計規格

**問題 6：每個關卡需要多大？**

參考：目前 Arena_Test 約 1 個螢幕大小

| 規模 | 說明 | 預估開發時間 |
|-----|------|------------|
| 小（1-2 螢幕） | 單一戰鬥區域 | 0.5 天 |
| 中（3-5 螢幕） | 多個戰鬥區 + 小路 | 1-2 天 |
| 大（探索型） | 分支路徑、隱藏區域 | 3-5 天 |

**問題 7：關卡美術風格？**

- [ ] 統一風格（全部森林）
- [ ] 區域變化（森林→洞穴→城堡）
- [ ] 暫時使用現有素材 placeholder

## 5. 技術限制說明

### 可行（低風險）
- 場景切換（走邊界 / 傳送門）
- 存檔點機制
- Hub + 多個獨立關卡
- 簡單的關卡地圖（靜態圖片）

### 需要額外工作（中風險）
- 無縫大地圖（需動態載入系統）
- 小地圖 Radar（需額外 UI 系統）
- 複雜的關卡生成（隨機地城）

### 需要先決條件（高風險）
- 美術資源（場景圖塊、背景）
- 關卡設計文件（敵人配置、物品放置）

## 6. 建議下一步

### 選項 A：最小可行（推薦）
1. 建立 `Town_Hub` 場景（安全區，有鐵匠）
2. 建立 `Field_Forest` 場景（野外戰鬥）
3. 實作「傳送門」切換機制
4. 在 Town_Hub 加入「前往野外」傳送點

**時間**：2-3 天
**價值**：玩家能體驗「城鎮→野外→戰鬥→回城」完整循環

### 選項 B：完整關卡
1. 設計完整世界地圖（5-8 個區域）
2. 建立所有場景
3. 實作複雜傳送網路
4. 加入探索要素（隱藏區域、收集品）

**時間**：1-2 週
**價值**：完整世界觀，但需大量美術/設計資源
