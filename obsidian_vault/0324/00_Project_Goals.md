## 專案名稱
異境 (Dream World)

## 版本 / 時間
Godot 4.x / 初版企劃 2026-03

---

## 核心願景

建立一個「短循環且具備成長感」的 ARPG 類別單人遊戲原型，使玩家可以在單一流程中體驗到：
- 與 NPC 對話 / 任務接取
- 與敵人戰鬥並獲得道具
- 道具納入背包並裝備武器
- 武器透過素材升級而具備可見成長

此循環必須「端到端可玩」：從進入遊戲到完成一次升級循環（對話→戰鬥→掉落→撿取→裝備→強化→存檔）可在多人 playtest 中重複驗證。

---

## 目標群體與核心體感

- **目標玩家**：偏好策略/動作混合的玩家，能容忍一定難度但期待「策略優勢」帶來的成就感。
- **核心體感**：以「策略優先 > 爽快體感 > 敵人複雜度」為導向。敵人行為應支持策略判定（優先目標、包圍、支援），同時把戰鬥手感（hit stop / FX）作為提升玩家滿意度的重點工作。

---

## 主要成功指標（KPI）

| 指標 | 目標值 | 驗證方式 |
|------|--------|----------|
| 功能完成率 | MVP 所列最小可交付全部通過驗收 | 功能驗收清單 |
| Playtest 可玩性 | 3 次內部 playtest 完成率 ≥ 80% | 玩家能完成至少一個完整循環 |
| 儲存穩定性 | 存檔／讀檔正確還原 | 5 次 Save/Load 循環驗證 |
| 單人開發可維持 | 無需額外長期外包 | 工時評估與進度追蹤 |

## 範圍界定（MVP）

### ✅ 包含（已完成 + 計劃完成）

| 系統 | 狀態 | 說明 |
|------|------|------|
| 玩家戰鬥 | ✅ | 攻擊狀態機、Combo、Dash |
| 敵人 AI | ✅ | Slime/Archer/Boar 三種類型 |
| 掉落 / 拾取 | ✅ | 掉落表、PickUpItem、自動撿取 |
| 背包 | ✅ | stackable + unique、20格 |
| 裝備 | ✅ | 數字鍵切換武器 |
| 存檔讀檔 | ✅ | SaveManager、版本遷移 |
| NPC 對話 | 🔄 | **下一階段重點** |
| 武器強化 | 🔄 | **下一階段重點** |
| 基礎 FX/HitStop | 🔄 | **下一階段重點** |

### ❌ 不包含（MVP 後期再擴充）

- 大型場景轉換 / Hub 系統（視進度決定）
- 大型 Boss 與多階段戰鬥（可放入中期）
- 完整群體 AI（可作為中期選項）
- 進階武器系統（鑲嵌/附魔）

## 技術約束與假設

- **引擎**：Godot 4.x
- **單人開發**：工時估算以 solo 開發為基準（程式 + 基本美術整合）
- **架構原則**：
  - 資源（Resource .tres）作為 Sources of Truth
  - Runtime 使用 Instance/DTO
  - Save 儲存最小可重建資料
  - 參考 `02_Tech_Spec_Notes.md` 規範

---

## 關鍵風險與緩解

| 風險 | 緩解策略 |
|------|----------|
| Save schema 破壞 | 採用 `save_version` + 遷移函式，提供向後相容性 |
| time_scale 副作用 | Hit stop 使用「局部時間暫停」而非全局 `Engine.time_scale` |
| 單人工時不足 | MVP-first 策略，先做最小可玩循環，placeholder art 快速驗證 |
| 模組耦合過高 | Autoload singletons + signals 作為系統間通訊界面 |
