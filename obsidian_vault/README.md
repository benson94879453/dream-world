# obsidian_vault 入口

> 最後更新: 2026-03-30

閱讀順序：

1. `sync_summary.md` - 目前狀態總覽
2. `planning/project_goals.md` - 專案目標
3. `planning/mvp_todo.md` - 任務清單與排程
4. `planning/tech_spec_notes.md` - 技術規範
5. `planning/implementation_order.md` - 實作順序與版本歷程
6. `planning/coding_habits.md` - 編碼習慣
7. `specs/input_keymap.md` - 按鍵與快捷鍵
8. `task_prompt.md` - 當前任務
9. `reviews/` - 驗收與回顧

三角色分工：

- `Task 指派者` - 拆任務、寫 prompt、維護總覽
- `實作者` - 依 prompt 實作，不改規格
- `驗收者` - 檢查修改、寫 review、決定通過與否

各角色閱讀檔案：

- `Task 指派者`
  - `README.md`
  - `sync_summary.md`
  - `planning/project_goals.md`
  - `planning/mvp_todo.md`
  - `planning/tech_spec_notes.md`
  - `planning/implementation_order.md`
  - `planning/coding_habits.md`
  - `specs/input_keymap.md`
  - `task_backlog.md`
  - 對應任務的 `reviews/*.md`

- `實作者`
  - `README.md`
  - `task_prompt.md`
  - `specs/input_keymap.md`
  - `task_prompt.md` 內列出的參考檔案

- `驗收者`
  - `README.md`
  - `task_prompt.md`
  - 實作者修改的檔案
  - `code_review_template.md`
  - 對應任務的歷史 `reviews/*.md`
  - `sync_summary.md`

文件角色：

- `workflow_template.md` - 正式協作模板
- `code_review_template.md` - 驗收模板
- `task_backlog.md` - 待指派任務池

歸檔文件：

- `archive/README.md` - 歷史文件索引
- `archive/workflow/workflow_legacy.md` - 舊版流程說明
- `archive/workflow/workflow_quickstart.md` - 舊版快速啟動
- `archive/reports/project_analysis_report.md` - 專案分析報告
- `archive/specs/weapon_strengthening_spec.md` - 舊版武器強化規格

若內容互相衝突，以 `README.md` 的閱讀順序和較新的更新日期為準。

歷史說明：`archive/` 與較早期的 `reviews/*.md` 會保留當時的命名與按鍵記錄，作為快照，不視為現行規範。
